# Update Docs Checklist

- [ ] Input deltas loaded from current planning and implementation artifacts
- [ ] Updated docs written only under approved `docs/*` destinations
- [ ] Modified docs align with current human policies
- [ ] Diataxis mode selected for new or materially changed docs
- [ ] Lifecycle, owner, and review metadata updated when required
- [ ] ADR or architecture deep-dive updates made when solutioning changes require them
- [ ] Planning-to-doc promotions record owner, lifecycle state, destination, and replacement or archive notes
- [ ] Parent and child README indexes updated for changed docs
- [ ] Relative links validated in modified docs
- [ ] Update report written to `{default_output_file}`
